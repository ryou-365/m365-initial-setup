m365-initial-setup

## 目的
本リポジトリは、Microsoft 365 管理センターの GUI 操作だけでは煩雑になりがちな初期設定を、PowerShell を用いて標準化・可視化することを目的としています。

「誰が構築しても、同じ強度で安全な認証基盤とサービス環境が完成する」状態を目指します。

## 基本方針
本リポジトリの手順は、以下の 3 つの原則に基づいて構成されています。

#### 実効性の高い「コピペと目視」の推奨
- コマンドを1行ずつ貼り付けて実行し、その場で出力を確認しながら進めることで、構築ミスや環境固有のエラーを即座に検知します。
#### 依存関係を考慮した段階的アプローチ
- 環境構築から始まり、認証基盤、ディレクトリ運用、各サービス設定へと、M365 の依存関係に基づいた順序で進めることで、手戻りのない構築を目指します。
#### 安全な作業環境の維持
- 管理者自身のロックアウトを防ぐためのアカウント確保や、レポートモードを利用した段階的なポリシー適用など、安全性を確保した手順を組み込んでいます。

## 構成概要
リポジトリは、構築のフェーズに合わせた段階的なマークダウンファイルで構成されています。

- 00_Initialization: 管理用実行環境のセットアップと、各サービスへのセキュアなエンドポイント接続の確立。
- 01_IdentityProt: 特権アクセスの管理と、コンテキストに基づいた動的な認証防御基盤の構築。
- 02_TenantGovernance: テナント全域に適用される「監視・資産・組織」の標準ルール策定と、自律的な運用を支える共通基盤の構築。
- 03_ExoSecurity: 通信経路の信頼性確保と、メッセージング環境における脅威排除のための保護設定。
- 04_SpoTeamsPolicy: コンテンツ共有の境界線定義と、共同作業領域におけるデータガバナンスの適用。
- 05_IntuneStandard: エンドポイントの信頼性評価と、組織リソースへアクセスするデバイスの構成標準化。

## 免責事項
本リポジトリの内容は、初期構築時のベストプラクティスを提示するものであり、組織固有の要件に応じて調整が必要です。

コマンドを実行する際は、必ず実行環境の権限および現在の設定を確認した上で自己責任で行ってください。

パスワードやテナント ID 等の機密情報は、決して本リポジトリ（Git 管理下）に保存しないでください。
